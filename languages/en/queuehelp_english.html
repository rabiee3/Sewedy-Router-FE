Queue

You can configure and manage the queues for IP-QoS. The lower numbers of precedence denote higher priority. It is recommended to use default pre-configured values of queues for getting optimum QoS

Queue Name Give the name to queue
Enable shows current status of queue, can be changed to Enable or disable
Interface Specify the interface for which the specified queue MUST exist
Queue Precedence Set the priority for the queue from dropdown menu
Traffic classes Comma-separated list of unsigned integers. Each list item identifies the set of traffic classes associated with this queue
Drop Algorithm select the Dropping algorithm used for this queue if congested
Schedule Algorithm Select one of the Scheduling Algorithm, which will be used by scheduler
Queue Weight Set the Weight for Queue in % ( in case of type WFQ/WRR)
Peak Shaping Rate Set the peak shaping rate for q in kbps
Location Where the Queue needs to be in (Egress/Ingress of interface)
Queue Length (in packets) This should be set in multiples of 8. Example- Setting value 7 will result in configuring 8 packets in system.
