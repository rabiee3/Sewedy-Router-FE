Paketfilterregel

Hinzufügen oder Bearbeiten von Paketfilterregeln zum Ermöglichen oder Verweigern der Pakete, die mit is übereinstimmen. IP-Version und Zielfelder sind Pflichtfelder, um eine Regel hinzuzufügen

Protokoll Wählen Sie aus dem Dropdown-Menü das Protokoll aus, um den Traffic zu steuern
Ursprungs IP-Addresse Der Traffic wird basierend auf der Quell-IP-Adresse gesteuert. Ein leeres Feld zeigt an, dass dieses Kriterium nicht zum Matching verwendet wird
Quellanschluss Quell-Portnummer
Source Port Bereich Der Verkehr wird basierend auf Ports zwischen Source Port und Source Port Range gesteuert. Source Port Range muss größer oder gleich Source Port sein
Ziel-IP-Adresse Der Traffic wird basierend auf der Ziel-IP-Adresse gesteuert. Ein leerer String zeigt an, dass dieses Kriterium nicht zum Matching verwendet wird
Zielhafen Ziel-Portnummer
Zielportbereich Der Traffic wird basierend auf den Ports zwischen Destination Port und Destination Port Range gesteuert. Zielanschlussbereich MUSS größer als oder gleich Zielanschluss sein
Quellschnittstelle Der Traffic wird basierend auf dem Source Interface gesteuert
Ziel-Schnittstelle Der Verkehr wird auf Basis der Zielschnittstelle gesteuert
IP Version IP-Protokollversion (z. B. 4 für IPv4 und 6 für IPv6)
Quell-MAC-Adresse Der Traffic wird basierend auf der MAC-Adresse gesteuert
Regelordnung Reihenfolge, in welcher Regel angewendet wird
Ziel Aktion, die für den Verkehr ausgeführt wird, der dem Eintrag "Regel" entspricht
Z.B. Mit der folgenden Regelkonfiguration wird der IPv4-Datenverkehr aus dem Gerät 192.168.1.2 mit dem Protokoll mit TCP übereinstimmen, die Zielport-Übereinstimmungen 8080 werden gelöscht.
Protokoll TCP
Ursprungs IP-Addresse 192.168.1.2
Zielhafen 8080
IP Version IPv4
Ziel DROP
